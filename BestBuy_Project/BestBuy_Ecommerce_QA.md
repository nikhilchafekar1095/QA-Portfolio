BestBuy E-commerce QA Project

Project Overview
This project focuses on manual testing of an e-commerce application similar to BestBuy. The aim is to validate major user flows and ensure the application works correctly from a customer point of view.

Modules Tested
- Login and Registration
- Product Search
- Product Details Page
- Add to Cart
- Checkout Process
- Payment Flow
- Order Confirmation

Testing Types Performed
- Functional Testing
- Regression Testing
- Smoke Testing
- Sanity Testing
- UI Testing
- Cross Browser Testing

Test Activities Performed
- Created and executed test cases for all major modules
- Identified and reported defects with proper steps to reproduce
- Performed regression testing after each build
- Verified UI behavior across different browsers
- Validated end-to-end checkout flow

Defects Reported
- Cart total calculation mismatch
- Search filter not working properly
- Checkout page validation issues
- UI alignment issues on product page

Tools Used
- JIRA for defect tracking
- Postman for API testing
- Chrome DevTools for debugging
- Excel for test case management

Outcome
The testing helped identify critical issues in checkout and search functionality, improving overall application stability and user experience.
